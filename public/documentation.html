MyMDB API Documentation

Welcome to the MyMDB API documentation. This API provides users with access to a comprehensive database of movies, directors, and genres. Users can create accounts, manage their profiles, and curate a list of their favorite movies. Below you will find the necessary information to start using the API.

Introduction

The MyMDB API is a RESTful service built using Node.js, Express, and MongoDB. It enables developers to:

  • Retrieve a list of all movies
  • Get detailed information about a specific movie
  • Access information about movie genres and directors
  • Create and manage user accounts
  • Allow users to maintain a list of their favorite movies

Each endpoint is accessed via standard HTTP methods such as GET, POST, PUT, and DELETE, and the API returns data in JSON format.

API Endpoints

Below is a list of available API endpoints and their respective functionalities:

Movies

  • GET /movies: Returns a list of all movies.
  • GET /movies/:movieId: Returns detailed information about a movie by movie Id.
  • GET /genres: Returns information about all genres and their descriptions.
  • GET /genres/:name: Returns information about a genre by name.
  • GET /directors: Returns information about all directors and their short bio.
  • GET /directors/:name: Returns information about a director by name.

Users

  • POST /users/register: Register a new user.
  • PUT /users/:userId: Update user information.
  • GET /users: Show all users and their favorite movies.
  • POST /users/:userId/movies/:movieId: Add a movie to a user's list of favorites.
  • DELETE /users/:userId/movies/:movieId: Remove a movie from a user's list of favorites.
  • DELETE /users/:userId: Deregister a user.

Mongoose Commands for API Endpoints

  • Users.findById(userId).select('-password'): Get a user by userId (GET /user/:userId)
  • Users.find().select('-password'): Get all users (GET /users)
  • Movie.find(): Get all movies (GET /movies)
  • Movie.findOne({ movieId: movieId }): Get a movie by movieId (GET /movies/:movieId)
  • Genre.find(): Get all genres (GET /genres)
  • Movie.find({ 'Genre.Name': genreName }): Get movies by genre name (GET /genres/:name)
  • Director.find(): Get all directors (GET /directors)
  • Movie.find({ 'Director.Name': directorName }): Get movies by director name (GET /directors/:name)
  • Users.findOne({ email: email }): Check if user exists during registration (POST /users/register)
  • Users.create({ name, email, password, birthday }): Create a new user (POST /users/register)
  • Users.findByIdAndUpdate(userId, { $set: updates }, { new: true }).select('-password') : Update a user's info (PUT /users/:userId)
  • Users.findOne({ Email: email }): Check if user exists before creation (POST /users)
  • Users.create({ userId, Password, Email, Birthday }): Create a new user (POST /users)
  • Users.findByIdAndUpdate(userId, { $addToSet: { FavoriteMovies: movieId } }, { new: true }).select('-password') : Add a movie to user's favorites (POST /users/:userId/movies/:movieId)
  • Users.findByIdAndUpdate(userId, { $pull: { FavoriteMovies: movieId } }, { new: true }).select('-password') : Remove a movie from user's favorites (DELETE /users/:userId/movies/:movieId)
  • Users.findByIdAndRemove(userId): Delete a user (DELETE /users/:userId)

Authentication

Authentication is handled using JSON Web Tokens (JWT). To access protected endpoints, users must include a valid JWT in the Authorization header of their requests. More details on how to obtain and use the token will be provided in future updates.

Usage Examples

Getting All Movies

GET /movies
  curl -X GET https://mymdb-api.herokuapp.com/movies

Registering a New User

POST /users/register
  curl -X POST https://mymdb-api.herokuapp.com/users \
  -H "Content-Type: application/json" \
  -d '{
    "email": "janedoe@example.com",
    "username": "janedoe"
  }'

Updating a User's Name

POST /users/userId
  curl -X POST https://mymdb-api.herokuapp.com/users \
  -H "Content-Type: application/json" \
  -d '{
    "name": "NewfirstName NewsecondName"
  }'

Error Handling

The API returns standard HTTP status codes to indicate the success or failure of an API request. Common status codes include:

  • 200 OK: The request was successful.
  • 201 Created: A new resource was successfully created.
  • 204 No Content: The request was successful.
  • 400 Bad Request: The request was invalid or cannot be served.
  • 401 Unauthorized: Authentication failed or the user does not have permissions for the requested operation.
  • 404 Not Found: The requested resource could not be found.
  • 500 Internal Server Error: An error occurred on the server.

Deployment

The MyMDB API is deployed on Heroku. The base URL for accessing the API is: https://mymdb-api.herokuapp.com. Ensure to replace localhost:8080 with this URL when making requests to the deployed API.
